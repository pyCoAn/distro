Feret Signature Tutorial

 

This tutorial gives a short introduction on how to use Feret Signatures to characterize the distribution of nanodisc samples within the pyCoAn interactive shell. It is complementary to the information provided in the Feret Signature preprint. After starting the pyCoAn shell using the command coan from your Linux prompt, the first step is to load the set of particle images. These will usually come from interactive or automated picking programs. The set is usually loaded into pyCoAn using the ImgStack command, which is an internal object representing image stacks. It accepts imagic (extensions img or hed) , hdf, or mrcs formats. If you have an image stack in mrcs format, you would load it into pyCoAn using:

 

     >>> stk = ImgStack(‘my_stack.mrcs’)

 

For the purpose of this tutorial, we use a preconfigured set of particle images from a simulated data set with a nanodisc diameter of 100 A. The classification and averaging step was already performed, resulting in 99 classes. The image stack can be loaded with the command:

 

     >>> stk = feret_tutorial_data()

 

Note: In our experience, a stack of 5000 to 10000 particle images is normally sufficient to get a reliable Feret signature. The ImgStack object behaves in some ways like a python list object and can be subjected to slicing. For example, if your input stack file has 50000 individual particle images, you can restrict stk to the first 5000 by specifying

 

     >>> stk = ImgStack(‘my_large_stack.hdf’)[:5000]

 

If you want to get fancy and select 5000 random particle images instead of the first 5000, use:

 

     >>> from random import sample

     >>> stk = ImgStack()

     >>> tmp = ImgStack(‘my_large_stack.hdf’)

     >>> for img in sample(tmp, 5000): stk.append(img)

     >>> del tmp

 

 

Run complete Feret signature evaluation

 

In the simplest case, you can run a complete evaluation with a single command, feret_eval. This command requires only one input parameters, the input particle image stack object. There are default values for all other parameters. They have been tweaked to give sensible results for a standard input but may have to be individually optimized for certain cases (see sections below). The complete list of options with detailed explanations can be obtained with the Help function via the following command:

 

     >>> Help(feret_eval)

 

For this tutorial, we will first use a default run but explicitly set the pixel size to the correct 4 A (default is 1 A):

 

>>> dct = feret_eval(stk, pix=4)

 

This command will automatically pop up visualization windows for the individual steps being performed including the class averages, the filtered class averages, the thresholded class averages, the segmented class averages, and the Feret signature. If the input stack has an attribute ferret_classavg (as the tutorial data has), it will omit the classification and averaging step. The output dct contains a dictionary with the statistics, the sizes of the individual particle images (out[sizes]), and some of the input parameters used. It will be saved automatically if the save parameter is set to True and can also be saved manually using:

 

     >>> save_obj(dct, ‘my_dct.pkl’)

 

Either can be read back into pyCoAn using:

 

     >>> dct = load_obj(‘my_dct.pkl’)

 

Where, of course, ‘my_dct’ is a placeholder for the actual name.

 

In some cases, it will be advantageous to refine the input parameters of the individual steps independently without rerunning the entire evaluation stack every time. The following sections of the tutorial describe how to do this in practice.

 

 

Classification and averaging

 

The main parameter to be concerned with for this step is num, which determines how many particles an individual class will have in average. Ideally, this number is kept small as not to average over different nanodisc sizes. However, if num is too small, then the noise level is too high and the determination of the of the individual Feret diameters becomes unreliable. For a standard, low SNR data set with thousands of particle images, the default (num = 50) seems to be a good compromise. For the tutorial, we will pretend that the tutorial data was not classified and averaged yet and run a classification with num = 5 (the minimum allowed)

 

     >>> avg = feret_classes(stk, num=5)

 

In this case, the resulting stack is not automatically visualized. It can be done using

 

     >>> avg.view()

 

You can also save the stack using

 

     >>> avg.write(‘my_avg.hdf’)

 

…and load it back in, possibly in a different session, using

 

     >>> avg = ImgStack(‘my_avg.hdf’)

 

The output of feret_classes acquires a new attribute feret_classavg, which is preserved upon saving and reloading in hdf format. This attribute will trigger to skip the class averaging step if the file is used as input to feret_eval.

 

 

Filtering and noise reduction

 

This step reduces the noise to a level were thresholding becomes reliable. The single parameter here is the filtering strength h and the default value is adequate for the vast majority of cases in our experience. For this example, we will reduce the strength from the default 1.5 to 0.5 just to show how it is done:

 

     >>> flt = feret_filter(avg, h=0.5)

     >>> flt.view()

 

The second command is optional for viewing the results.

 

The output of feret_filter acquires a new attribute feret_filtered, which is preserved upon saving and reloading in hdf format. This attribute will trigger to skip the filtering step if the file is used as input to feret_eval (it will also have the feret_classavg attribute and skip the class averaging as well).

 

 

Thresholding, segmentation, and signature extraction

 

The subsequent steps do not require any adjustments so from here on, we can switch back to feret_eval, which will automatically skip the feret_classes and feret_filter steps:

 

     >>> dct_avg = feret_eval(flt, pix=4)

 

You will notice that the mean and median will be more or less preserved while the standard deviation is reduced in comparison to the original calculation. This is a direct effect of the additional averaging.

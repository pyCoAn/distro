MEPSi Tutorial

 

This tutorial gives a short introduction on how to use MEPSi, the Membrane-Embedded Protein Simulator within the pyCoAn interactive shell. It is complementary to the information provided in the MEPSi preprint. After starting the pyCoAn shell using the command coan from your Linux prompt, the first step is to generate a list of coordinate files (currently only PDB format is supported) that will be used to populate the simulation. The molecules need to be oriented with the extracellular domain pointing into the negative Z direction (away from you in Chimera). The center of the transmembrane domain should be at the origin [0,0,0]. And aligned with the Z axis. To generate the list, you can use the glob command, which is preloaded in pyCoAn. For example, if you want to generate a list with all PDB files starting with “A” in the current directory, you would use:

 

     >>> lst = glob(‘A*.pdb’)

     >>> lst.sort()

 

The second step is optional but can make subsequent steps more streamlined because the order of lists generated by the glob command is not consistent. Lists with single entries are also allowed. For the purpose of this tutorial, a preconfigured and correctly oriented set of coordinate files, corresponding to the open and closed conformation of integrin (see here for background) can be loaded with the command:

 

     >>> lst = mepsi_tutorial_data()

 

MEPSi is designed with modularity in mind, so some steps in the tutorial can be skipped (those will be denoted as “This step is optional”). However, the first step will always be generating the density from the list of coordinate files.

 

Note: For full functionality, tomo3d, which drives the reconstruction calculations, needs to be installed and accessible from within pyCoAn. If you do not have it already, you can obtain it here. To check whether it is correctly installed and set up for MEPSi use:

 

     >>> check_access(‘tomo3d’)

 

If the command returns False you either do not have tomo3d installed or it is not properly accessible from within pyCoAn as tomo3d. An imod installation is not strictly required but is highly recommended for viewing the various density and tilt series files. You can check for proper installation and set up using the check_access command.

 

 

Generate density

 

This is achieved using the command sim_membrane_prots. The command requires three input parameters. (1) the input coordinate file list, (2) the radius of the sphere the molecules will be placed on and (3) the average distance between the molecules. There are defaults for all other parameters. The complete list of options with detailed explanations can be obtained with the Help function via the following command:

 

     >>> Help(sim_membrane_prots)

 

For this tutorial, we will generate a simulated tomograms of an extracellular vesicle with embedded integrin receptors in the closed and open conformations. We start by building a sphere of 400 Angstrom radius with integrins place every 200 Angstroms apart. Instead of the tight default box size, we will generate the density within a box of 500 x 500 x 400 pixels. Because the default pixel size is 4 Angstroms, this will generate a box that fully contains the sphere and the embedded integrins as well as some extra padding. In addition, we will set the ratio so that the probability of picking the open conformation is twice the probability of picking a closed conformation.

 

>>> den = sim_membrane_prots(lst, 400, 200, ratio=[1,2], outsize=[500,500,400])

 

The density generated through this command can either be used directly to calculate a tilt series or a solvent model can be added before calculating the tilt series.

 

Note: You can save the density for use in subsequent sessions with something like:

 

     >>> den.write(“my_density.hdf”)

 

…and read it back into a pyCoAn session with:

 

     >>> den = Map(“my_density.hdf”)

 

 

Add solvent model

 

This step is optional. To apply the default solvent model, simply use:

 

     >>> sol = sim_add_solvent(den)

 

As a somewhat more complex example, we increase the variance of the solvent level and decrease the graininess of the variations:

 

     >>> sol = sim_add_solvent(den, grain=50, var=0.24)

 

To check the full range of options available, use the Help function.

 

 

Calculating a basis tilt series

 

This step generates a ‘basis tilt series’ that can be used as a basis to subsample, add contrast transfer effects, or noise. We would typically choose the finest sampling and the largest tilt range that make sense for the planned exploration. For example, if we want to compare a tomogram generated with 1º tilt increment and ±45º tilt range with a tomogram generated with 3º tilt increment and ±60º tilt range, we would generate a basis tilt series using 1º tilt increment (parameter ‘step’) and ±60º tilt range (parameter ‘tilts’). The two test scenarios can then be generated by subsampling the basis tilt series (see tomo_simulator).

 

     >>> ts = tilt_simulator(sol, tilts=[-60,60], step=1)

 

This is usually the most time-consuming step. Note: If you skipped the solvent modeling, use den instead of sol. We can inspect the tilt series using

 

     >>> ts.imod()

 

Note: You can save the tilt series for use in a later session using something like:

 

     >>> ts.write(“my_tilt_series.hdf”)

 

Make sure to use the hdf extension, otherwise the metadata will be incomplete. Load the tilt series with:

 

     >>> ts = ImgStack(“my_tilt_series.hdf”)

 

 

Apply a contrast transfer function

 

This step is optional and allows the application of contrast transfer function parameters to the basis tilt series. Parameters include defocus, voltage, phase shift and amplitude contrast. Use the Help function to obtain a list of available parameters. Here, we simply introduce a defocus of 5 microns. The defocus gradient due to the tilt geometry is automatically accounted for by the algorithm (in this case, the field of view is small enough so that only a single defocus value is necessary).

 

     >>> ctf = sim_apply_ctf(ts, dm=5)

 

We can inspect the results of the operation with

 

     >>> ctf.imod()

 

 

Apply noise model

 

This step is optional and applies a noise model to the input tilt series. The model is based on mixing Gaussian and Laplacian components to introduce skewness and better match the texture of the noise typically encountered in experimental tilt series. Here, we apply the standard model with a signal-to-noise ratio of 0.65:

 

     >>> noi = sim_apply_noise (ctf, snr=0.65)

 

Note: If you skipped the CTF application step, use ts instead of ctf. Inspect results with:

 

     >>> noi.imod()

 

 

Simulate tomograms

 

This step is used to generate individual tomograms. It uses a tilt series (either plain or with applied CTF and/or noise) and applies filtering operations to generate tomograms with different tilt parameters. To generate a tomogram with a specific tilt range, use the maxtilts parameter. Note that the range needs to be within the range of the basis tilt series to take effect (otherwise it will just truncate to the values of te basis tilt series). To modify the tilt increment, use the sample parameter. This indicates the relative sampling in respect to the basis tilt series. For example, sample = 2 takes every second tilt image. So, if the basis tilt series is calculated with 1º tilt increments, sample = 2 will result in a tilt series with 2º tilt increments, If the basis tilt series was calculated with 1.5º tilt increments, sample = 2 will result in a tilt series with 3º tilt increments. For this tutorial, we will generate a tomogram with 3º tilt increments and a tilt range from -60º to +50º. We use the same thickness as the original density to facilitate comparison.

 

>>> tomo = tomo_simulator(ts, 400, maxtilts=[-60,50], sample=3)

 

Note: If a CTF model was applied (and no noise model) use ctf instead of ts, if a noise model was applied, use noi.

 

 

Obtain metrics

 

To obtain the comparison metrics, use the original density before the basis tilt series was calculated (within this tutorial, this will be either sol or den if a solvent model was not added). Before using these measures, it is important to normalize the two densities:

 

     >>> sol.norm()

     >>> tomo.norm()

 

Now, the metrics can be calculated.

 

     >>> mse(tomo, -sol)

     >>> psnr(tomo, -sol)

     >>> mssim(tomo, -sol)

 

mse and psnr are quick to calculate but mssim takes quite a while. For visualization, imod is probably the most useful. The following command will open an imod process with the unperturbed density and the simulated tomogram loaded:

 

     >>> tomo.imod(-sol)

 

Note: We use -sol here because sol is positive contrast and tomo is negative contrast. If no solvent model was added, use -den instead of -sol.

 

 

Dual tilt geometry

 

To simulate a dual tilt geometry, one would rotate the simulated density after sim_add_solvent (or right after sim_membrane_prots if no solvent model is required) around the Z axis. This can be achieved with the rotate function:

 

     >>> duo = rotate(den, z=90)

 

The subsequent steps would be applied to both densities den and duo separately to generate two separate simulated tomograms tomo1 and tomo2. Note that the chosen parameters do not have to match. For example, it is possible to test the effect of adding a ±45º tilt series with 5º tilt increment to a ±60º tilt series with 2º tilt increment after rotation. Once the two tomograms are generated, simply rotate back the second one by using rotate with the negative rotation angle and add it to the first tomogram:

 

     >>> comb = tomo1 + rotate(tomo2, z=-90)

 

Note: This simple procedure only works if (1) X and Y match, (2) the rotation angle is exactly 90º, and (3) the thickness of the tomograms match. It is possible to use more general parameters, but it will complicate the process to an extent that is out of the scope of this tutorial.
